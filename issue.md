# Reporting issues

Use the GitHub Issues tab for bugs and feature requests.

## Before opening an issue

- Check existing issues for duplicates.
- For bugs: confirm on latest `master` and with backend + frontend running as in README.

## What to include

**Bug**

- Short title and steps to reproduce.
- What you expected vs what happened.
- Environment: Node version, OS, MongoDB (local/Atlas).
- Backend/frontend logs or errors if relevant.

**Feature / change**

- Clear description and why it would help.
- Optional: how youâ€™d implement it (API, UI).

## Where it happens

- Backend (API, auth, DB, CLI).
- Frontend (page/component, route).
- Both / integration.
